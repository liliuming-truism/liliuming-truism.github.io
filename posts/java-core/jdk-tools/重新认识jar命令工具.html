重新认识jar命令工具

LiLiuMing9/21/25...About 3 minJavaJDKJDK-Tool

什么是 jar 命令？

jar 命令是 JDK 自带的 Java 归档工具，用于创建、查看和管理 JAR（Java Archive）文件。JAR 文件基于 ZIP 和 ZLIB 压缩格式，是 Java 应用程序打包和分发的标准格式。

jar 命令语法结构

jar [操作模式] [选项] [目标文件] [源文件/目录]

jar 命令的语法设计类似于 Unix 的 tar 命令，使用起来简洁高效。每次使用时必须指定一个操作模式，配合相应的选项和参数。

主要操作模式

参数长选项功能描述
-c--create创建新的归档文件
-t--list列出归档文件中的内容
-u--update更新已存在的 JAR 文件
-x--extract从归档文件中提取文件
-d--describe-module打印模块描述符或自动模块名称

通用选项参数

基础选项

参数长选项功能描述
-f FILE--file=FILE指定归档文件名称
-v--verbose显示详细操作信息
-C DIR切换到指定目录进行操作

创建/更新模式专用选项

参数长选项功能描述
-e CLASSNAME--main-class=CLASSNAME指定可执行 JAR 的主类
-M--no-manifest不创建 MANIFEST.MF 文件
-p--module-path指定模块路径
--module-version=VERSION指定模块版本

实战示例

1. 创建 JAR 文件

基础创建：

jar -cf myapp.jar -C bin/ .
  • -c：创建模式
  • -f：指定输出文件名
  • -C bin/：切换到 bin 目录
  • .：打包当前目录下所有文件

创建可执行 JAR：

jar -cfe myapp.jar com.example.Main -C bin/ .
  • -e com.example.Main：指定主类，生成可执行 JAR

详细输出创建：

jar -cvf myapp.jar -C bin/ .
  • -v：显示打包过程的详细信息

2. 查看 JAR 文件内容

列出所有文件：

jar -tf myapp.jar

详细列出文件信息：

jar -tvf myapp.jar

3. 提取 JAR 文件

提取所有文件：

jar -xf myapp.jar

提取特定文件：

jar -xf myapp.jar META-INF/MANIFEST.MF

4. 更新 JAR 文件

添加新文件：

jar -uf myapp.jar -C bin/ NewClass.class

批量更新：

jar -uvf myapp.jar -C bin/ .

高级用法

模块化 JAR 操作

查看模块信息：

jar -d -f module-app.jar

创建模块化 JAR：

jar --create --file=module-app.jar --main-class=com.example.App --module-version=1.0 -C classes/ .

多版本 JAR 支持

jar --create --file=multiversion.jar --release 8 -C classes-8/ . --release 11 -C classes-11/ .

最佳实践

1. 目录结构建议

project/
├── src/          # 源代码
├── classes/      # 编译后的 .class 文件
├── lib/          # 依赖库
└── dist/         # 打包输出目录

2. 常用打包脚本

#!/bin/bash
# 编译
javac -d classes -cp "lib/*" src/**/*.java

# 打包
jar -cvfe dist/myapp.jar com.example.Main -C classes/ .

# 验证
jar -tf dist/myapp.jar | head -10

3. MANIFEST.MF 自定义

# 创建自定义 manifest 文件
echo "Main-Class: com.example.Main" > manifest.txt
echo "Class-Path: lib/dependency.jar" >> manifest.txt

# 使用自定义 manifest 打包
jar -cfm myapp.jar manifest.txt -C classes/ .

常见问题与解决方案

Q: 如何解决"没有主清单属性"错误？

A: 使用 -e 参数指定主类或手动创建 MANIFEST.MF 文件。

Q: 如何在 JAR 中包含依赖库？

A: 可以将依赖 JAR 解压后一起打包，或使用 Maven/Gradle 的 fat jar 插件。

Q: 如何验证 JAR 文件的完整性？

A: 使用 jar -tf 命令检查文件列表，或使用 java -jar 测试运行。

总结

jar 命令是 Java 开发中不可或缺的工具，掌握其各种用法能够大大提高开发效率。从简单的文件打包到复杂的模块化应用构建，jar 命令都能胜任。

提示： 现代 Java 项目通常使用 Maven 或 Gradle 等构建工具自动处理 JAR 打包，但了解底层的 jar 命令原理对于问题排查和深入理解 Java 生态系统仍然很有价值。
