How to understand JLS、JSR、JEP？

LiLiuMing7/28/25...About 3 minJavaJDKJSRJLSJEP

How to understand JLS、JSR、JEP？

JLS (Java Language Specification)

Java 语言规范

定义

JLS 是 Java 编程语言的官方规范文档，定义了 Java 语言的语法、语义和行为。

作用

  • 📋 权威标准：定义 Java 语言的所有特性和规则
  • 🔧 编译器指南：为 JVM 实现者和编译器开发者提供标准
  • 📚 参考文档：解决语言特性的歧义和争议

内容包括

  • 词法结构（关键字、标识符、字面量）
  • 类型系统（基本类型、引用类型）
  • 表达式和语句
  • 类和接口定义
  • 异常处理机制
  • 并发和线程模型

版本对应

JLS 1  → Java 1.0
JLS 2  → Java 1.1  
JLS 3  → Java 1.2-1.4
JLS 4  → Java 5
JLS 5  → Java 6-7
JLS 8  → Java 8
JLS 11 → Java 11
JLS 17 → Java 17

JSR (Java Specification Request)

Java 规范请求

定义

JSR 是 Java 社区流程（JCP）中用于提出、开发和批准 Java 平台新特性或 API 的正式文档。

流程

  1. 提交阶段：任何 JCP 成员都可以提交 JSR
  2. 审查阶段：专家委员会评估提案
  3. 开发阶段：专家组开发规范和参考实现
  4. 投票阶段：JCP 执行委员会投票决定是否采纳

重要的 JSR 示例

JSR 14  → 泛型 (Generics)
JSR 175 → 注解 (Annotations) 
JSR 201 → 枚举类型 (Enums)
JSR 310 → 日期时间 API (java.time)
JSR 335 → Lambda 表达式
JSR 376 → Java 模块系统 (Project Jigsaw)

状态类型

  • Draft：草案阶段
  • Public Review：公开审查
  • Final：最终版本
  • Maintenance：维护阶段
  • Withdrawn：已撤回

JEP (JDK Enhancement Proposal)

JDK 增强提案

定义

JEP 是 OpenJDK 社区用于提出、跟踪和协调 JDK 中重大变更的流程和文档格式。

特点

  • 🎯 具体实现：关注具体的 JDK 实现细节
  • 📊 项目管理：跟踪开发进度和状态
  • 🔄 迭代改进：支持渐进式的功能开发

JEP 生命周期

Draft → Candidate → Funded → Completed
  ↓
Withdrawn (可能的状态)

重要的 JEP 示例

JEP 286 → 局部变量类型推断 (var 关键字)
JEP 305 → instanceof 模式匹配
JEP 359 → Records (记录类)
JEP 384 → switch 表达式
JEP 395 → sealed 类
JEP 409 → 密封类

三者关系对比

方面JLSJSRJEP
性质规范文档标准化流程增强提案
范围语言规范Java 平台标准JDK 实现
管理者OracleJCPOpenJDK
更新频率每个大版本按需提交持续进行
关注点语言定义标准制定具体实现

实际应用场景

开发者视角

// 当你想了解某个语法的精确定义时 → 查阅 JLS
// 例如：泛型的类型擦除机制

// 当你想了解某个 API 的标准定义时 → 查阅相关 JSR  
// 例如：JSR 310 了解 java.time API

// 当你想了解新特性的实现细节时 → 查阅相关 JEP
// 例如：JEP 286 了解 var 关键字的实现
  1. JEPs propose and develop experimental ideas to the point where they could be specified. Not all JEPs come to fruition.
  2. JSRs take mature ideas (e.g. resulting from a JEP), and produce a new specification, or modifications to an existing specification. Not all JSRs come to fruition.
  3. A specification is a common work product of a JSR. (Others include source code of interfaces, and reference implementations.) The JLS is an example of a specification. Others include the JVM specification (JVMS), the Servlet and JSP specifications, the EJB specifications and so on.

这三个概念共同构成了 Java 生态系统的标准化和发展框架，理解它们有助于更好地掌握 Java 语言的发展脉络和技术细节。
