Spring IOC容器概述与核心接口设计

Spring IOC容器概述与核心接口设计

IOC容器基本概念

控制反转(IOC)和依赖注入(DI)的概念

1.1 什么是控制反转（IOC）

控制反转（Inversion of Control）是一种设计原则，它将对象的创建和依赖关系的管理从应用程序代码中分离出来，交给外部容器来处理。

传统方式：

public class UserService {
private UserDao userDao;

    public UserService() {
        // 对象自己创建依赖
        this.userDao = new UserDaoImpl();
    }
}

IOC方式：

public class UserService {
private UserDao userDao;

    // 通过构造函数注入依赖
    public UserService(UserDao userDao) {
        this.userDao = userDao;
    }
}

1.2 依赖注入（DI）

依赖注入（Dependency Injection）是实现IOC的一种具体方式，它通过外部容器将依赖对象注入到目标对象中。

Spring支持三种主要的依赖注入方式：

  1. ​ 构造函数注入：通过构造函数参数注入依赖
  2. ​ Setter方法注入：通过setter方法注入依赖
  3. ​ 字段注入：直接在字段上注入依赖（通过注解）

Spring IOC容器的作用和优势

  1. 降低耦合度：对象之间的依赖关系由容器管理

  2. 提高可测试性：便于进行单元测试和集成测试

  3. 增强可维护性：配置和代码分离，易于维护

  4. 支持AOP：为面向切面编程提供基础

  5. 统一管理：对象生命周期统一管理

Spring IOC容器的整体架构

Spring IOC容器采用分层架构设计，从底层的Bean管理到上层的应用上下文，每一层都有明确的职责分工。整个架构可以分为以下几个层次：

  • 核心接口层：定义核心接口和契约
  • 扩展功能层：提供企业级功能扩展
  • 基础实现层：提供基本的Bean管理功能
  • 辅助实现层：为实现Spring IOC提供辅助

核心接口层（Core Interface Layer）

这一层定义了Spring IOC容器的核心抽象和契约：

BeanFactory接口
  • Spring IOC容器的根接口
  • 定义了获取Bean、检查Bean存在性、判断Bean作用域等基本操作
  • 采用延迟加载策略，只有在获取Bean时才进行实例化
public interface BeanFactory {
    // 获取Bean实例，对应的Bean不存在时会抛出异常 
    Object getBean(String name) throws BeansException;
    <T> T getBean(String name, Class<T> requiredType) throws BeansException;
    <T> T getBean(Class<T> requiredType) throws BeansException;
    
    // 获取延迟初始化的Bean或者可选的Bean
    ObjectProvider<T> getBeanProvider(Class<T> requiredType);
    ObjectProvider<T> getBeanProvider(ResolvableType type);
    
    // 检查Bean是否存在
    boolean containsBean(String name);
    
    // 检查Bean的作用域
    boolean isSingleton(String name) throws NoSuchBeanDefinitionException;
    boolean isPrototype(String name) throws NoSuchBeanDefinitionException;
    
    // 获取Bean的类型
    Class<?> getType(String name) throws NoSuchBeanDefinitionException;
    Class<?> getType(String name, boolean allowFactoryBeanInit) throws NoSuchBeanDefinitionException;
    
    // 判断Bean的类型是否与目标类型一致
    boolean isTypeMatch(String name, Class<?> typeToMatch) throws NoSuchBeanDefinitionException;
    boolean isTypeMatch(String name, ResolvableType typeToMatch) throws NoSuchBeanDefinitionException;
    
    // 获取Bean的别名
    String[] getAliases(String name);
}
ApplicationContext接口
  • 继承自BeanFactory，提供更多企业级功能
  • 增加了国际化、事件发布、资源加载等功能
  • 采用预加载策略，在容器启动时就实例化所有单例Bean
public interface ApplicationContext extends EnvironmentCapable, 
    ListableBeanFactory, HierarchicalBeanFactory,
    MessageSource, ApplicationEventPublisher, ResourcePatternResolver {
    
    // 获取应用上下文ID
    String getId();
    
    // 获取应用名称
    String getApplicationName();
    
    // 获取显示名称
    String getDisplayName();
    
    // 获取启动时间戳
    long getStartupDate();
    
    // 获取父上下文
    ApplicationContext getParent();
    
    // 获取自动装配Bean工厂
    AutowireCapableBeanFactory getAutowireCapableBeanFactory() 
        throws IllegalStateException;
}
BeanDefinition接口
  • 描述Bean的元数据信息
  • 包含Bean的类名、作用域、依赖关系、初始化方法等配置信息
  • 是Spring IOC容器管理Bean的基础数据结构
public interface BeanDefinition extends AttributeAccessor, BeanMetadataElement {
    // 作用域常量
    String SCOPE_SINGLETON = ConfigurableBeanFactory.SCOPE_SINGLETON;
    String SCOPE_PROTOTYPE = ConfigurableBeanFactory.SCOPE_PROTOTYPE;
    
    // 角色常量
    int ROLE_APPLICATION = 0;
    int ROLE_SUPPORT = 1;
    int ROLE_INFRASTRUCTURE = 2;
    
    // Bean类名
    void setBeanClassName(String beanClassName);
    String getBeanClassName();
    
    // 作用域
    void setScope(String scope);
    String getScope();
    
    // 懒加载
    void setLazyInit(boolean lazyInit);
    boolean isLazyInit();
    
    // 依赖关系
    void setDependsOn(String... dependsOn);
    String[] getDependsOn();
    
    // 自动装配候选
    void setAutowireCandidate(boolean autowireCandidate);
    boolean isAutowireCandidate();
    
    // 主要候选
    void setPrimary(boolean primary);
    boolean isPrimary();
    
    // 构造函数参数
    ConstructorArgumentValues getConstructorArgumentValues();
    
    // 属性值
    MutablePropertyValues getPropertyValues();
    
    // 表示此Bean是否是抽象的，抽象Bean不能被实例化，只能作为其他Bean的父类Bean
    boolean isAbstract();
    
    // 表示是否作为一个后备的Bean，当主要得到Bean定义不可用时，使用此定义
    boolean isFallback();
    	
    // 表示是否可以作为自动装配的候选者，注意，此方法不会影响按名称自动注入
    boolean isAutowireCandidate();
    
    // 初始化方法
    void setInitMethodName(String initMethodName);
    String getInitMethodName();
    
    // 销毁方法
    void setDestroyMethodName(String destroyMethodName);
    String getDestroyMethodName();
}

扩展接口层（Extended Interface Layer）

这一层在核心接口基础上提供了更多专门化的功能

HierarchicalBeanFactory
/**
 * HierarchicalBeanFactory扩展了BeanFactory，提供了父子容器的层次结构支持
 * 
 * 核心职责：
 * 1. 建立Bean工厂的父子关系，支持容器层次结构
 * 2. 实现Bean查找的层次化机制（先查本地，再查父容器）
 * 3. 提供本地Bean存在性检查，避免向上查找
 * 4. 支持容器隔离，子容器可以访问父容器Bean，反之不行
 * 5. 为Web应用的多层容器架构提供基础支持
 * 
 * 设计理念：
 * - 分层隔离：不同层次的Bean定义相互隔离
 * - 向上查找：子容器找不到Bean时自动向父容器查找
 * - 单向依赖：子容器依赖父容器，父容器不依赖子容器
 * 
 * 典型应用场景：
 * - Spring MVC中的Root ApplicationContext和Servlet ApplicationContext
 * - 模块化应用中的核心容器和业务模块容器
 * - 插件系统中的宿主容器和插件容器
 * 
 * 核心方法：
 * - getParentBeanFactory(): 获取父容器引用
 * - containsLocalBean(): 检查本地是否包含指定Bean
 * 
 * 查找策略：
 * 1. 首先在当前容器中查找Bean
 * 2. 如果找不到且存在父容器，则向父容器查找
 * 3. 递归向上查找直到找到Bean或到达根容器
 */
public interface HierarchicalBeanFactory extends BeanFactory {
    
    // 获取父Bean工厂，支持容器层次结构
    @Nullable
    BeanFactory getParentBeanFactory();
    
    // 检查本地Bean工厂是否包含指定Bean，不向上查找
    boolean containsLocalBean(String name);
}
ConfigurableBeanFactory
/**
 * ConfigurableBeanFactory扩展了HierarchicalBeanFactory和SingletonBeanRegistry，
 * 提供了Bean工厂的配置和管理功能
 * 
 * 核心职责：
 * 1. 提供Bean工厂的完整配置能力（类加载器、类型转换、属性编辑等）
 * 2. 管理Bean后处理器的注册和执行
 * 3. 支持自定义作用域的注册和管理
 * 4. 提供Bean依赖关系的管理和追踪
 * 5. 支持表达式解析和值解析功能
 * 6. 管理Bean的生命周期（创建、初始化、销毁）
 * 7. 提供Bean工厂的运行时配置修改能力
 * 
 * 配置能力：
 * - 类加载器配置：setBeanClassLoader、setTempClassLoader
 * - 类型转换配置：setConversionService、setTypeConverter
 * - 属性编辑配置：addPropertyEditorRegistrar、registerCustomEditor
 * - 表达式解析配置：setBeanExpressionResolver
 * - 值解析配置：addEmbeddedValueResolver
 * 
 * 扩展机制：
 * - BeanPostProcessor：Bean实例化前后的处理
 * - Scope：自定义Bean作用域
 * - PropertyEditor：属性类型转换
 * - BeanExpressionResolver：表达式解析
 * 
 * 生命周期管理：
 * - 依赖关系追踪：registerDependentBean、getDependentBeans
 * - Bean销毁管理：destroyBean、destroyScopedBean、destroySingletons
 * - 创建状态管理：setCurrentlyInCreation、isCurrentlyInCreation
 * 
 * 使用场景：
 * - ApplicationContext内部配置BeanFactory
 * - 自定义容器实现
 * - 容器扩展和插件开发
 * - 测试环境中的容器配置
 */
public interface ConfigurableBeanFactory extends HierarchicalBeanFactory, SingletonBeanRegistry {
    
    // 作用域常量
    String SCOPE_SINGLETON = "singleton";
    String SCOPE_PROTOTYPE = "prototype";
    
    // 父容器配置
    void setParentBeanFactory(BeanFactory parentBeanFactory) throws IllegalStateException;
    
    // 类加载器配置
    void setBeanClassLoader(@Nullable ClassLoader beanClassLoader);
    ClassLoader getBeanClassLoader();
    
    // 类型转换配置
    void setConversionService(@Nullable ConversionService conversionService);
    ConversionService getConversionService();
    
    // Bean后处理器管理
    void addBeanPostProcessor(BeanPostProcessor beanPostProcessor);
    int getBeanPostProcessorCount();
    
    // 作用域管理
    void registerScope(String scopeName, Scope scope);
    String[] getRegisteredScopeNames();
    
    // 依赖关系管理
    void registerDependentBean(String beanName, String dependentBeanName);
    String[] getDependentBeans(String beanName);
    
    // 生命周期管理
    void destroyBean(String beanName, Object beanInstance);
    void destroySingletons();
}
ListableBeanFactory
/**
 * ListableBeanFactory扩展了BeanFactory，提供了Bean的枚举和批量操作功能
 * 
 * 核心职责：
 * 1. 提供Bean定义的统计和枚举功能
 * 2. 支持按类型批量获取Bean实例
 * 3. 支持按注解类型查找Bean
 * 4. 提供Bean名称的各种查找方式
 * 5. 支持Bean的批量操作和管理
 * 6. 为依赖注入提供候选Bean的发现机制
 * 
 * 枚举功能：
 * - getBeanDefinitionCount(): 获取Bean定义总数
 * - getBeanDefinitionNames(): 获取所有Bean定义名称
 * - containsBeanDefinition(): 检查Bean定义是否存在
 * 
 * 类型查找：
 * - getBeanNamesForType(): 根据类型获取Bean名称数组
 * - getBeansOfType(): 根据类型获取Bean实例映射
 * - 支持包含/排除非单例Bean的选项
 * - 支持延迟初始化控制选项
 * 
 * 注解查找：
 * - getBeanNamesForAnnotation(): 根据注解获取Bean名称
 * - getBeansWithAnnotation(): 根据注解获取Bean实例
 * - findAnnotationOnBean(): 在指定Bean上查找注解
 * 
 * 设计特点：
 * - 只读接口：不提供修改Bean定义的方法
 * - 批量操作：支持一次性获取多个相关Bean
 * - 类型安全：提供泛型支持的类型查找
 * - 灵活控制：支持各种查找条件和选项
 * 
 * 典型应用场景：
 * - 自动装配时查找候选Bean
 * - 按类型注入多个Bean实例
 * - AOP切面的目标Bean发现
 * - 事件监听器的自动注册
 * - 插件系统的组件发现
 */
public interface ListableBeanFactory extends BeanFactory {
    
    // Bean定义统计
    boolean containsBeanDefinition(String beanName);
    int getBeanDefinitionCount();
    String[] getBeanDefinitionNames();
    
    // 按类型查找Bean名称
    String[] getBeanNamesForType(ResolvableType type);
    String[] getBeanNamesForType(@Nullable Class<?> type);
    String[] getBeanNamesForType(@Nullable Class<?> type, boolean includeNonSingletons, boolean allowEagerInit);
    
    // 按类型获取Bean实例
    <T> Map<String, T> getBeansOfType(@Nullable Class<T> type) throws BeansException;
    <T> Map<String, T> getBeansOfType(@Nullable Class<T> type, boolean includeNonSingletons, boolean allowEagerInit);
    
    // 按注解查找
    String[] getBeanNamesForAnnotation(Class<? extends Annotation> annotationType);
    Map<String, Object> getBeansWithAnnotation(Class<? extends Annotation> annotationType);
    <A extends Annotation> A findAnnotationOnBean(String beanName, Class<A> annotationType);
}
AutowireCapableBeanFactory
/**
 * AutowireCapableBeanFactory扩展了BeanFactory，提供了Bean的自动装配和生命周期管理功能
 * 
 * 核心职责：
 * 1. 提供多种自动装配策略（byName、byType、constructor等）
 * 2. 支持Bean的完整生命周期管理（创建、装配、初始化、销毁）
 * 3. 提供外部Bean实例的装配能力
 * 4. 支持依赖解析和注入
 * 5. 集成Bean后处理器到Bean生命周期
 * 6. 提供Bean创建的编程式API
 * 
 * 自动装配模式：
 * - AUTOWIRE_NO(0): 不自动装配
 * - AUTOWIRE_BY_NAME(1): 按名称自动装配
 * - AUTOWIRE_BY_TYPE(2): 按类型自动装配  
 * - AUTOWIRE_CONSTRUCTOR(3): 按构造函数自动装配
 * - AUTOWIRE_AUTODETECT(4): 自动检测装配模式（已废弃）
 * 
 * Bean生命周期支持：
 * - createBean(): 创建Bean实例
 * - autowireBean(): 装配已存在的Bean
 * - configureBean(): 配置Bean（装配+应用Bean属性）
 * - initializeBean(): 初始化Bean
 * - destroyBean(): 销毁Bean
 * 
 * 后处理器集成：
 * - applyBeanPostProcessorsBeforeInitialization(): 初始化前处理
 * - applyBeanPostProcessorsAfterInitialization(): 初始化后处理
 * 
 * 依赖解析：
 * - resolveDependency(): 解析指定的依赖
 * - 支持复杂的依赖描述符
 * - 支持类型转换和集合注入
 * 
 * 设计特点：
 * - 策略模式：支持多种装配策略
 * - 模板方法：定义Bean处理的标准流程
 * - 扩展性强：通过后处理器支持自定义逻辑
 * 
 * 典型应用场景：
 * - Spring框架内部的Bean创建和装配
 * - 集成第三方框架的Bean管理
 * - 测试环境中的Bean mock和装配
 * - 动态Bean创建和配置
 */
public interface AutowireCapableBeanFactory extends BeanFactory {
    
    // 自动装配模式常量
    int AUTOWIRE_NO = 0;
    int AUTOWIRE_BY_NAME = 1;
    int AUTOWIRE_BY_TYPE = 2;
    int AUTOWIRE_CONSTRUCTOR = 3;
    
    // Bean创建
    <T> T createBean(Class<T> beanClass) throws BeansException;
    Object createBean(Class<?> beanClass, int autowireMode, boolean dependencyCheck);
    
    // Bean装配
    void autowireBean(Object existingBean) throws BeansException;
    Object autowire(Class<?> beanClass, int autowireMode, boolean dependencyCheck);
    
    // Bean配置
    Object configureBean(Object existingBean, String beanName) throws BeansException;
    void autowireBeanProperties(Object existingBean, int autowireMode, boolean dependencyCheck);
    void applyBeanPropertyValues(Object existingBean, String beanName);
    
    // Bean初始化
    Object initializeBean(Object existingBean, String beanName) throws BeansException;
    Object applyBeanPostProcessorsBeforeInitialization(Object existingBean, String beanName);
    Object applyBeanPostProcessorsAfterInitialization(Object existingBean, String beanName);
    
    // Bean销毁
    void destroyBean(Object existingBean);
    
    // 依赖解析
    @Nullable
    Object resolveDependency(DependencyDescriptor descriptor, @Nullable String requestingBeanName);
}
ConfigurableListableBeanFactory
/**
 * ConfigurableListableBeanFactory是Spring BeanFactory接口层次的顶级集成接口，
 * 集合了ListableBeanFactory、AutowireCapableBeanFactory和ConfigurableBeanFactory的所有功能
 * 
 * 核心职责：
 * 1. 提供最完整的BeanFactory功能集合
 * 2. 支持Bean定义的注册、查询、修改和删除
 * 3. 提供高级的依赖解析和自动装配配置
 * 4. 支持Bean工厂的预初始化和优化
 * 5. 提供Bean定义的冻结和元数据缓存管理
 * 6. 支持复杂的Bean查找和依赖关系管理
 * 
 * 功能集成：
 * - 继承ListableBeanFactory: Bean枚举和批量操作
 * - 继承AutowireCapableBeanFactory: 自动装配和生命周期管理
 * - 继承ConfigurableBeanFactory: 配置和扩展能力
 * 
 * 高级配置功能：
 * - ignoreDependencyType(): 忽略特定类型的依赖注入
 * - ignoreDependencyInterface(): 忽略特定接口的依赖注入
 * - registerResolvableDependency(): 注册可解析的依赖
 * - isAutowireCandidate(): 检查Bean是否为自动装配候选
 * 
 * 性能优化功能：
 * - freezeConfiguration(): 冻结Bean定义配置，提高运行时性能
 * - isConfigurationFrozen(): 检查配置是否已冻结
 * - clearMetadataCache(): 清除元数据缓存
 * - preInstantiateSingletons(): 预实例化所有单例Bean
 * 
 * Bean定义管理：
 * - getBeanDefinition(): 获取Bean定义
 * - getBeanNamesIterator(): 获取Bean名称迭代器
 * - 支持Bean定义的动态修改和查询
 * 
 * 设计特点：
 * - 功能完备：集成了所有BeanFactory相关功能
 * - 高度可配置：提供了丰富的配置选项
 * - 性能优化：支持配置冻结和预初始化
 * - 扩展友好：为框架扩展提供了完整的API
 * 
 * 典型应用场景：
 * - ApplicationContext的内部BeanFactory实现
 * - 企业级应用的容器配置
 * - 框架集成和扩展开发
 * - 高性能要求的生产环境
 * - 复杂的Bean依赖关系管理
 */
public interface ConfigurableListableBeanFactory 
        extends ListableBeanFactory, AutowireCapableBeanFactory, ConfigurableBeanFactory {
    
    // 依赖注入控制
    void ignoreDependencyType(Class<?> type);
    void ignoreDependencyInterface(Class<?> ifc);
    void registerResolvableDependency(Class<?> dependencyType, @Nullable Object autowiredValue);
    boolean isAutowireCandidate(String beanName, DependencyDescriptor descriptor);
    
    // Bean定义访问
    BeanDefinition getBeanDefinition(String beanName) throws NoSuchBeanDefinitionException;
    Iterator<String> getBeanNamesIterator();
    
    // 性能优化
    void clearMetadataCache();
    void freezeConfiguration();
    boolean isConfigurationFrozen();
    void preInstantiateSingletons() throws BeansException;
}

基础实现层

这一层提供了接口的核心实现：

AbstractBeanFactory
/**
 * AbstractBeanFactory是BeanFactory接口的抽象基类，提供了IOC容器的核心框架
 * 
 * 核心职责：
 * 1. 定义Bean获取的标准流程模板（doGetBean方法）
 * 2. 管理单例Bean的缓存机制
 * 3. 处理父子容器的层次结构
 * 4. 管理Bean后处理器（BeanPostProcessor）
 * 5. 支持不同作用域（singleton、prototype等）
 * 6. 提供类型转换和属性编辑功能
 * 
 * 设计模式：
 * - 模板方法模式：定义获取Bean的标准流程，具体创建逻辑由子类实现
 * - 策略模式：支持不同的作用域策略
 * 
 * 关键抽象方法：
 * - createBean(): 留给子类实现具体的Bean创建逻辑
 * - containsBeanDefinition(): 检查是否包含Bean定义
 * - getBeanDefinition(): 获取Bean定义信息
 */
public abstract class AbstractBeanFactory extends FactoryBeanRegistrySupport 
        implements ConfigurableBeanFactory {
    
    // 核心模板方法：定义获取Bean的标准流程
    protected <T> T doGetBean(String name, Class<T> requiredType, Object[] args, boolean typeCheckOnly);
    
    // 抽象方法：由子类实现具体的Bean创建逻辑
    protected abstract Object createBean(String beanName, RootBeanDefinition mbd, Object[] args);
    
    // 单例缓存管理、Bean后处理器管理、作用域支持等核心功能...
}
AbstractAutowireCapableBeanFactory
/**
 * AbstractAutowireCapableBeanFactory继承AbstractBeanFactory，专门负责Bean的创建、属性注入和初始化
 * 
 * 核心职责：
 * 1. 实现Bean的完整生命周期管理（实例化→属性注入→初始化→销毁）
 * 2. 提供多种自动装配策略（byName、byType、constructor等）
 * 3. 解决循环依赖问题（三级缓存机制）
 * 4. 集成Bean后处理器到Bean生命周期各个阶段
 * 5. 支持Aware接口回调（BeanNameAware、BeanFactoryAware等）
 * 6. 处理初始化方法和销毁方法的调用
 * 
 * Bean创建流程：
 * 1. 实例化Bean（createBeanInstance）
 * 2. 处理循环依赖（提前暴露Bean引用）
 * 3. 属性注入（populateBean）
 * 4. 初始化Bean（initializeBean）
 * 5. 注册销毁回调
 * 
 * 设计特点：
 * - 实现了AutowireCapableBeanFactory接口的所有自动装配功能
 * - 提供了Bean创建的完整模板方法
 * - 支持复杂的依赖注入场景
 */
public abstract class AbstractAutowireCapableBeanFactory extends AbstractBeanFactory 
        implements AutowireCapableBeanFactory {
    
    // 实现父类的抽象方法：Bean创建的核心实现
    protected Object createBean(String beanName, RootBeanDefinition mbd, Object[] args);
    
    // Bean创建的详细流程
    protected Object doCreateBean(String beanName, RootBeanDefinition mbd, Object[] args);
    
    // 属性注入的核心方法
    protected void populateBean(String beanName, RootBeanDefinition mbd, BeanWrapper bw);
    
    // Bean初始化方法
    protected Object initializeBean(String beanName, Object bean, RootBeanDefinition mbd);
}
DefaultListableBeanFactory
/**
 * DefaultListableBeanFactory是Spring IOC容器的默认实现，集成了所有BeanFactory相关接口的功能
 * 
 * 核心职责：
 * 1. 实现BeanDefinitionRegistry接口，管理Bean定义的注册、查询、修改
 * 2. 实现ListableBeanFactory接口，提供Bean的枚举和批量操作功能
 * 3. 支持按类型查找Bean（getBeansOfType、getBeanNamesForType）
 * 4. 支持按注解查找Bean（getBeansWithAnnotation、getBeanNamesForAnnotation）
 * 5. 提供预实例化功能（preInstantiateSingletons）
 * 6. 管理Bean之间的依赖关系
 * 7. 支持Bean定义的继承和覆盖
 * 
 * 存储结构：
 * - beanDefinitionMap: 存储Bean定义的核心映射表
 * - beanDefinitionNames: 按注册顺序存储Bean名称
 * - singletonObjects: 单例Bean缓存
 * 
 * 功能特点：
 * - 是ApplicationContext内部使用的默认BeanFactory
 * - 提供了最完整的Bean管理功能
 * - 支持复杂的Bean查找和依赖解析
 * - 线程安全的Bean定义管理
 */
public class DefaultListableBeanFactory extends AbstractAutowireCapableBeanFactory 
        implements ConfigurableListableBeanFactory, BeanDefinitionRegistry {
    
    // Bean定义存储
    private final Map<String, BeanDefinition> beanDefinitionMap = new ConcurrentHashMap<>(256);
    
    // Bean定义注册方法
    public void registerBeanDefinition(String beanName, BeanDefinition beanDefinition);
    
    // 按类型获取Bean
    public <T> Map<String, T> getBeansOfType(Class<T> type);
    
    // 预实例化所有单例Bean
    public void preInstantiateSingletons();
}
AbstractApplicationContext
/**
 * AbstractApplicationContext是ApplicationContext的核心抽象实现，提供了应用上下文的完整生命周期管理
 * 
 * 核心职责：
 * 1. 定义ApplicationContext的标准刷新流程（refresh方法的12个步骤）
 * 2. 集成事件发布机制（ApplicationEventPublisher）
 * 3. 支持国际化功能（MessageSource）
 * 4. 提供资源访问功能（ResourceLoader）
 * 5. 管理环境配置（Environment和Profile）
 * 6. 协调BeanFactory和ApplicationContext的功能
 * 7. 支持优雅关闭和生命周期管理
 * 
 * refresh()方法的12个步骤：
 * 1. prepareRefresh() - 准备刷新
 * 2. obtainFreshBeanFactory() - 获取BeanFactory
 * 3. prepareBeanFactory() - 准备BeanFactory
 * 4. postProcessBeanFactory() - 后处理BeanFactory
 * 5. invokeBeanFactoryPostProcessors() - 调用BeanFactory后处理器
 * 6. registerBeanPostProcessors() - 注册Bean后处理器
 * 7. initMessageSource() - 初始化消息源
 * 8. initApplicationEventMulticaster() - 初始化事件多播器
 * 9. onRefresh() - 刷新特定上下文
 * 10. registerListeners() - 注册监听器
 * 11. finishBeanFactoryInitialization() - 完成BeanFactory初始化
 * 12. finishRefresh() - 完成刷新
 * 
 * 设计特点：
 * - 模板方法模式：定义标准的上下文刷新流程
 * - 组合模式：内部组合DefaultListableBeanFactory
 * - 观察者模式：支持事件监听机制
 */
public abstract class AbstractApplicationContext extends DefaultResourceLoader 
        implements ConfigurableApplicationContext {
    
    // 内部持有的BeanFactory
    private DefaultListableBeanFactory beanFactory;
    
    // ApplicationContext的核心方法：刷新容器
    public void refresh() throws BeansException, IllegalStateException;
    
    // 抽象方法：由子类实现具体的BeanFactory刷新逻辑
    protected abstract void refreshBeanFactory() throws BeansException, IllegalStateException;
    
    // 获取内部BeanFactory
    public abstract ConfigurableListableBeanFactory getBeanFactory() throws IllegalStateException;
}

AnnotationConfigApplicationContext
/**
 * AnnotationConfigApplicationContext是基于Java注解配置的ApplicationContext实现

 * 核心职责：

 * 7. 提供编程式Bean注册功能
 * 
 * 核心组件：
 * - AnnotatedBeanDefinitionReader: 读取注解Bean定义
 * - ClassPathBeanDefinitionScanner: 扫描类路径组件
 * - ConfigurationClassPostProcessor: 处理@Configuration类
 * 
 * 支持的注解：

 * - 需要灵活配置的项目

 * 继承层次：
 * AnnotationConfigApplicationContext
 *   → GenericApplicationContext
 *     → AbstractApplicationContext
 */
public class AnnotationConfigApplicationContext extends GenericApplicationContext 
        implements AnnotationConfigRegistry {

容器启动流程概览

ApplicationContext启动时序图

容器启动主要阶段

  1. prepareRefresh() - 准备刷新
    • 设置启动时间和活跃标志
    • 初始化属性源
    • 验证必需的属性
  2. obtainFreshBeanFactory() - 获取BeanFactory
    • 创建DefaultListableBeanFactory实例
    • 加载BeanDefinition
    • 返回配置好的BeanFactory
  3. prepareBeanFactory() - 准备BeanFactory
    • 设置BeanFactory的类加载器
    • 设置表达式解析器
    • 添加属性编辑器注册器
  4. postProcessBeanFactory() - 后处理BeanFactory
    • 允许子类对BeanFactory进行进一步设置
    • 通常在Web环境中注册特定的后处理器
  5. invokeBeanFactoryPostProcessors() - 调用BeanFactory后处理器
    • 执行BeanDefinitionRegistryPostProcessor
    • 执行BeanFactoryPostProcessor
    • 处理配置类和组件扫描
  6. registerBeanPostProcessors() - 注册Bean后处理器
    • 注册实现了BeanPostProcessor接口的Bean
    • 这些后处理器将在Bean实例化时被调用
  7. initMessageSource() - 初始化MessageSource
    • 初始化国际化资源
    • 如果没有定义则使用默认的DelegatingMessageSource
  8. initApplicationEventMulticaster() - 初始化事件广播器
    • 初始化ApplicationEventMulticaster
    • 用于发布应用事件
  9. onRefresh() - 刷新特定上下文
    • 模板方法，允许子类进行特定的刷新操作
    • 例如Web容器中启动内嵌的Servlet容器
  10. registerListeners() - 注册监听器
    • 注册静态指定的监听器
    • 注册ApplicationListener类型的Bean
  11. finishBeanFactoryInitialization() - 完成BeanFactory初始化
    • 初始化剩余的单例Bean
    • 这是最耗时的阶段
  12. finishRefresh() - 完成刷新
    • 初始化LifecycleProcessor
    • 发布ContextRefreshedEvent事件
    • 注册到JMX（如果需要）
